# ElyProxy

Windows-клиент для VLESS-подписок на базе Xray-core. Приложение поднимает локальный SOCKS5-прокси, умеет подключаться к VLESS-серверам, работать с подписками, включать системный прокси Windows с разделением трафика и запускать ElyTun для полного TUN-режима.

## Возможности

- Подключение к VLESS-серверам через Xray-core
- Поддержка VLESS-ссылок из подписок и ручного ввода `vless://`
- Локальный SOCKS5-прокси: `127.0.0.1:1080`
- Системный прокси Windows с PAC-правилами
- ElyTun: TUN/VPN-режим на базе Xray TUN inbound
- Разделение трафика: выбранные домены идут напрямую, остальное через прокси
- Редактируемые правила разделения трафика в настройках
- Автозапуск вместе с Windows
- Автоподключение к последнему выбранному серверу
- Автопереподключение при обрыве соединения
- Автообновление подписок по выбранному интервалу
- TCP-пинг серверов и отображение задержки
- Proxy GET-пинг через временный локальный SOCKS5, как проверка реального прокси-маршрута
- ElyTurbo: проверка популярных сервисов через сервер и выбор лучшего кандидата
- ElyHard: полный real-session тест с повторными открытиями сервисов через прокси
- Профили серверов с импортом и экспортом `.elyproxy`
- Экспорт встроенного файла правил `OmegaRules_auto_switch.sorl` для ZeroOmega
- Быстрые ссылки на расширения ZeroOmega для Firefox и Chrome
- Возможность скрыть или показать панель логов
- Сохранение подписок, профилей, ручных серверов и настроек

## Системный Прокси

Режим **Системный прокси** включает PAC-конфигурацию в настройках Windows. Это позволяет направлять через прокси только нужный трафик, а часть доменов открывать напрямую.

По умолчанию напрямую идут домены:

```text
*.ru
*.рф
*.xn--p1ai
*.su
*.com.ru
*.edu.ru
```

Правила можно изменить во вкладке **Настройки** → **Разделение трафика**. Каждое правило указывается с новой строки.

Важно:

- системный прокси активируется только после подключения к серверу;
- при отключении от сервера системный прокси выключается;
- если режим был включён до отключения, он автоматически активируется снова после следующего подключения.

## ElyTun

**ElyTun** — режим полного системного туннеля. Он добавляет в конфиг Xray отдельный `tun` inbound, создаёт TUN-интерфейс Windows и направляет системный IPv4-трафик через выбранный сервер. Локальные/private-адреса остаются напрямую, чтобы не ломать доступ к локальной сети.

Особенности:

- требуется запуск ElyProxy от имени администратора;
- ElyTun и **Системный прокси** взаимоисключаются;
- ElyTun не запускается параллельно с другим `xray.exe` или активным VPN/TUN-клиентом;
- Radmin VPN, Happ, Wintun, WireGuard, OpenVPN и похожие адаптеры входят в стоп-лист ElyTun;
- системные Windows-туннели вроде Teredo не блокируют запуск ElyTun;
- при включении ElyTun приложение перезапускает текущий Xray с TUN inbound;
- при выключении ElyTun Xray перезапускается обратно в обычный SOCKS5-режим;
- нужен Xray `26.4.0` или новее;
- MTU и IPv6 не вынесены в интерфейс: ElyProxy использует безопасные внутренние значения;
- DNS для TUN берётся из активного сетевого адаптера Windows и идёт напрямую, чтобы избежать таймаутов DoH через прокси.

В конфиге используется `autoOutboundsInterface: "auto"`, чтобы исходящее подключение самого Xray к серверу не зацикливалось обратно в TUN.

## Установка

### Готовая Сборка

1. Перейдите в [Releases](https://github.com/CrackKO/ElyProxy/releases)
2. Скачайте архив `ElyProxy-v1.1.0-win-x64.zip`
3. Распакуйте в любую папку
4. Если в архиве нет папки `xray`, выполните пункты 5-6.
5. Скачайте [Xray-core](https://github.com/XTLS/Xray-core/releases) (`Xray-windows-64.zip`)
6. Из архива Xray поместите `xray.exe`, `geoip.dat`, `geosite.dat` и `wintun.dll` в папку `xray/` рядом с `ElyProxy.exe`
7. Запустите `ElyProxy.exe`

Если Xray-core не входит в сборку, скачайте `Xray-windows-64.zip` из релизов Xray-core и поместите рядом с приложением:

```text
xray/xray.exe
xray/geoip.dat
xray/geosite.dat
xray/wintun.dll
```

Требуется Windows и [.NET 8 Runtime](https://dotnet.microsoft.com/download/dotnet/8.0/runtime), если приложение собрано не как self-contained. Для ElyTun нужен Xray `26.4.0` или новее.

### Сборка Из Исходников

```bash
git clone https://github.com/CrackKO/ElyProxy.git
cd ElyProxy
dotnet restore
dotnet build
dotnet run
```

Release-сборка:

```bash
dotnet publish -c Release -r win-x64 --self-contained false -o publish
```

## Использование

1. Откройте ElyProxy.
2. Перейдите во вкладку **Подписки**.
3. Введите название и URL подписки.
4. Нажмите **Добавить**.
5. Перейдите во вкладку **Серверы**.
6. Выберите сервер и нажмите **Подключить**.
7. При необходимости включите **Системный прокси** или **ElyTun** на вкладке **Серверы**.

Для приложений, где прокси нужно указать вручную:

```text
socks5://127.0.0.1:1080
```

Кнопка **Пинг** на вкладке **Серверы** использует режим, выбранный в настройках:

- **TCP пинг** — быстрый замер подключения до адреса и порта сервера;
- **Proxy GET** — параллельный запуск временных Xray-проверок и GET-запрос через прокси, чтобы проверить реальную работу маршрута. URL по умолчанию: `https://youtube.com`.
- **ElyTurbo** — проверка Discord, Telegram, YouTube, Spotify, GitHub и Gmail. Сначала учитывается количество доступных сервисов, затем средняя задержка. Лучший сервер автоматически выбирается в списке.
- **ElyHard** — полный real-session тест: для каждого сервера запускается временный Xray, затем Discord, Telegram, YouTube, Spotify, GitHub и Gmail открываются через этот прокси в несколько попыток. Рейтинг учитывает стабильные сервисы, успешные попытки и задержку.

## Настройки

Во вкладке **Настройки** доступны:

- запуск вместе с Windows;
- подключение к последнему серверу;
- переподключение при обрыве;
- показ или скрытие логов;
- локальный SOCKS5-порт;
- тип пинга для кнопки **Пинг**;
- URL для Proxy GET-пинга;
- количество потоков для Proxy GET, ElyTurbo и ElyHard;
- интервал автообновления подписок;
- правила разделения трафика для системного прокси.

## Хранение Данных

Данные хранятся в:

```text
%AppData%/ElyProxy/
```

Основные файлы:

- `subscriptions.json` — подписки и серверы из подписок;
- `profiles.json` — пользовательские профили;
- `settings.json` — настройки, ручные серверы, правила системного прокси и состояние ElyTun.

## Примечания

- Для работы системного прокси приложение меняет пользовательские настройки Windows Internet Settings.
- При выключении системного прокси приложение пытается восстановить предыдущее значение `AutoConfigURL`.
- Если приложение было завершено аварийно, системный прокси можно отключить вручную в настройках Windows.
- Для ElyTun приложение запускает Xray TUN inbound и меняет системные маршруты Windows, поэтому нужны права администратора.
- Если ElyTun был аварийно завершён и сеть ведёт себя странно, перезапустите ElyProxy от имени администратора и нажмите **Отключить** либо перезагрузите Windows.
